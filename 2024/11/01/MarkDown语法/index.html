MarkDown语法

发表于 2024-11-01 分类于 tool

MarkDown基础语法

标题

使用符号（#/=），最多支持六级标题，最好在符号后加上空格

  • #  一级标题
  • ##  二级标题
  • ###  三级标题

段落

语法如下

  • 如果行与行之间没有空行则会被视为一段，若之间有有空行则视为不同段落
  • 空行是指行内什么都没有，或只有空格和制表符
  • 如果想在段内直接换行则需要在上一行的结尾插入两个以上的空格然后回车
  • 当超过80个字符或以（。！?）等结尾时建议换行，或URL较长时换行

列表

有序列表语法如下

  • 数字序号 + 英文句号 + 空格 + 列表内容
    1. 我是第一行
    2. 我是第二行
    3. ……..
  • 无序列表如下
    */+/- + 空格 + 列表内容

列表嵌套语法如下

  • - 第一层列表
    tab + 第二层列表
    tab+ tab + 第三层列表

  • 第一层列表
    • 第二层列表
      • 第三层列表
        • …..

图片插入

图片插入语法文字链接如下

  • ![图片替代文字](图片替代地址)
  • [链接文字](链接地址)
    • 引用连接：

      [链接文字][链接标记]
      ……
      在底部:
      [链接标记]: 链接地址

引用链接

  • 链接标记可以有字母、数字、空格和标点符号
  • 链接标记不区分大小写
  • 定义的链接内容可以放在当前文件的任意位置、建议放在页尾巴
  • 当链接地址为网络地址时要以http/https开头，否则会被标识为本地地址

行内代码块

  • 行内代码引用要使用`包裹，语法如下
    使用`cd..` -> CD ..

  • 代码块.TAB键或者4个空格开头

      def test_print():
      pass

使用规范

引用

语法如下

引用由>+ 引用内容来标记

  • 多行引用也可以在每一行的开有都插入>
  • 在引用中可以嵌套引用
  • 在引用中可以使用其它MarkDown语法
  • 段落与换行的格式在引用中也是可以适用的
    • 示例

      例子1

      正常的句子

      多行引用第一行
      第二行

    • 引用嵌套

      引用嵌套 >

      被嵌套部分 > >

GFM语法

删除线

  • ~~被删除的文字~~
    被删除的文字

表情符号

  • 表情符号 :表情代码:
    :smile:
    :laughing:
    :+1:

自动链接

  • 由<>包裹的URL地址被自动识别并解析为超链接
    百度https://www.baidu.com

表格

  • 表格语法如下
    \| 表头1 \| 表头2 \| 表头3 \| \|---- \|---- \| ----\| \| 内容1 \| 内容2\| 内容3\|
    表头1 表头2 表头3
    内容1 内容2 内容3
    内容1 内容2 内容3
  • 默认对齐

    • 左对齐【默认】 :

    • 右对齐 -：

    • 中间对齐 :-:

      中间对齐 |:—-:|:—-:|:—-:|

      表头1 表头2 表头3
      内容1 内容2 内容3
      内容1 内容2 内容3

任务列表

  • 任务列表以-+ 开头，由[+空格/X+]组成

  • X 可以有大小写,看编辑器支持，推荐小写，当中括号内为x时为选中状态

    • 吃

    • 喝

    • 玩

      • 吃
        • 吃鱼
        • 吃饭

锚点（书签）

[锚点描述](#锚点名)

  • 锚点名建议使用字母和数字，锚点名是区分大小写的
  • 在锚点中不能含有空格也不能含有特殊字符
